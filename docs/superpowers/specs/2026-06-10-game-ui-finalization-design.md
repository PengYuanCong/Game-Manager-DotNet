# 遊戲網站前端收尾設計

## 目標

將現有 ASP.NET Core MVC 網站整理成可反覆使用的 ARAM Mayhem 玩家工具，而不是宣傳型首頁或制式 AI 儀表板。保留既有資料、CRUD、搜尋、推薦與登入流程，集中改善共用視覺語言、資訊層級、互動回饋與響應式排版。

## 設計判讀

- 產品類型：ARAM Mayhem 配裝與海克斯決策工具
- 主要使用者：已經進入對局、需要快速比較選項的中文玩家
- 視覺方向：海克斯戰情室 + 普羅助手
- 設計參數：
  - Design variance：6
  - Motion intensity：4
  - Visual density：7
- 技術基礎：沿用 Razor Views、Bootstrap 5、Bootstrap Icons 與原生 CSS/JavaScript
- 不新增前端框架或動畫套件

## 核心原則

1. AI 推薦是主要任務，英雄、海克斯與裝備是決策資料來源。
2. 玩家在一個視窗內應能看清主要操作，不用先閱讀網站說明。
3. 普羅是引導角色，不是每頁重複的大型裝飾。
4. 卡片只用於可選擇、可比較或可操作的資料項目。
5. 不使用紫藍 AI 漸層、滿版網格、玻璃卡片牆或無意義發光。
6. 所有互動保留鍵盤焦點、清楚 hover、pressed 與 loading 狀態。

## 視覺系統

### 色彩

- 基底：深墨藍與帶綠的黑色，避免純黑。
- 主要強調：舊金色，用於主要操作、標題重點與選取狀態。
- 功能色：青綠色，只用於可互動提示、成功狀態與資料關聯。
- 稀有度色彩保留既有語意：
  - 白銀：冷灰藍
  - 黃金：琥珀金
  - 稜彩：低飽和紫
- 危險操作維持紅色，但只出現在刪除、錯誤與登出。

### 字體

- 中文介面沿用本機可用的繁體中文字型堆疊，避免額外字型網路依賴。
- 英數標籤使用較窄的系統無襯線字體與 tabular numbers。
- 大標題控制在兩行內；資料頁標題不使用 landing page 尺寸。
- 小標籤使用句子式文字，不堆疊全大寫英文。

### 形狀與材質

- 頁面區塊以留白、分隔線與背景層次組織，不把每區包成卡片。
- 卡片圓角維持 4-6px；按鈕與輸入框使用相同半徑規則。
- 主面板可使用輕微切角，但不在每個元件重複。
- 背景只保留局部紋理、地圖或角色圖，不再滿版鋪網格。

## 共用框架

### 導覽列

- 第一層主要任務：
  - AI 推薦
  - 英雄
  - 海克斯
  - 裝備
- 第二層工具：
  - 公式
  - 精彩操作
  - 個人資料
- 桌面版以一條緊湊導覽呈現，主要任務權重較高。
- 手機版維持 Bootstrap collapse，但增加清楚分組與完整點擊區。
- 品牌區以遊戲控制器與簡短站名呈現，不做大型發光框。
- 登入、使用者與登出維持原有行為。

### 頁面框架

- 共用最大寬度與左右 gutter。
- 所有主要頁面使用一致的 page header：
  - 模式或功能標籤
  - 頁面標題
  - 一句任務說明
  - 主要操作或搜尋工具
- 內容載入前保持版面尺寸，避免切頁時看到未套樣式內容。

## 首頁

首頁改成直接可理解的產品入口，不再使用五張環形流程卡。

### 第一視窗

- 左側：產品定位與主要 CTA「開始 AI 推薦」。
- 次要 CTA：「瀏覽英雄」。
- 右側：實際玩法流程預覽，使用三個階段：
  1. 選擇英雄
  2. 比較本輪三個海克斯
  3. 取得海克斯與裝備方向
- 普羅位於流程區，作為提示與推薦來源。
- 背景可沿用現有遊戲圖，但調整裁切、遮罩與對比，避免文字被背景干擾。

### 次要入口

- 提供英雄、海克斯與裝備三個資料入口。
- 不使用三張完全相同的行銷卡片。
- 採一個主入口搭配兩個次入口的非對稱排列。

## AI 推薦頁

- 保留既有英雄選擇、階段、三選一海克斯與備註流程。
- 左側輸入區改成分段任務：
  1. 鎖定英雄
  2. 選擇對局階段
  3. 帶入本輪三個海克斯
  4. 補充對局狀況
- 右側在未產生結果時展示簡短決策預覽，不留大片空白。
- 結果區以「推薦選擇、原因、裝備方向、注意事項」排序。
- 收藏與帶入操作使用一致的動詞和按鈕層級。
- 行動裝置上先顯示輸入，再顯示結果。

## 英雄、海克斯與裝備資料頁

### 共通

- 統一搜尋與篩選工具列。
- 搜尋是主要指令，新增、匯入與 CRUD 只對管理員顯示。
- 卡片 hover 使用 160-220ms 的微抬升與邊框亮度變化。
- 卡片高度由內容決定，不為了整齊強迫所有文字區同高。
- 圖片有穩定尺寸與 fallback，不因圖片載入造成版面跳動。

### 英雄

- 英雄頭像、名稱、定位與人工判斷是第一層資訊。
- 推薦海克斯與裝備摘要使用可掃讀標籤，不輸出長段落。
- 詳情頁將打法、站位、弱點與人工備註分區呈現。

### 海克斯

- 預設依評級排序。
- 稀有度、套裝與屬性標籤保留不同語意色。
- 屬性抽屜保持固定寬度，不展開時不得佔滿頁面高度。
- 套裝浮動按鈕保留，但降低裝飾性，避免遮擋卡片。

### 裝備

- 維持價格由低到高與既有屬性篩選。
- 裝備介紹拆成名稱、基礎屬性、效果與資料來源。
- 已選裝備與儲存組合保持跨篩選狀態。
- 編輯與刪除只對管理員顯示。

## 公式、精彩操作與個人資料

- 套用同一頁面框架、導覽、表單、按鈕與空狀態。
- 不重寫公式計算或影片功能。
- 公式頁優先修正下拉選單、結果層級與比較區。
- 精彩操作頁以影片縮圖與操作分類為主，不做宣傳式 hero。
- 個人資料以最近五筆活動、收藏與裝備組合為主。

## 動態與互動

- 頁面進場：opacity + translateY，120-180ms。
- 卡片 hover：translateY(-3px)，160-220ms。
- 按鈕按下：translateY(1px) 或 scale(0.98)。
- 所有動畫只使用 transform 與 opacity。
- 遵守 `prefers-reduced-motion`。
- 不使用循環發光、視差滾動、磁吸游標或長時間載入動畫。

## 響應式規則

- 1440px 以上維持內容最大寬度，不無限拉伸。
- 992-1199px 導覽與工具列允許換行，但不可重疊。
- 768-991px 將雙欄頁改為單欄，主要操作保持在上方。
- 767px 以下：
  - 主標題縮至可在兩至三行內閱讀
  - 搜尋與篩選使用全寬控制
  - 卡片操作按鈕維持至少 44px 點擊高度
  - 不出現水平捲軸

## 可存取性

- 保留 skip link 與語意化 `nav`、`main`、`section`、`article`。
- 表單 label 不以 placeholder 取代。
- 焦點狀態清楚且符合色彩對比。
- 圖片提供有意義的替代文字。
- 裝飾圖片使用空 alt 或 CSS background。
- 色彩不是唯一的稀有度與狀態識別方式。

## 實作範圍

### 必做

- 共用 `_Layout.cshtml`
- 共用 `site.css` 與必要的共用 JavaScript
- 首頁
- AI 推薦首頁與結果狀態
- 英雄、海克斯、裝備列表頁
- 公式、精彩操作與個人資料的共用視覺收斂
- 登入、註冊與 404 的設計 token 對齊

### 不做

- 不更換 MVC 架構
- 不新增 React、Tailwind、GSAP 或其他前端套件
- 不更動資料庫結構
- 不改寫推薦演算法或 RAG 資料
- 不重新爬取外部資料
- 不直接複製 Riot、OP.GG 或 Mobalytics 的版型與素材

## 驗收標準

1. 首頁第一視窗可立即辨認 AI 推薦是核心。
2. 導覽在 1280px、1024px、768px 與 390px 不重疊。
3. 主要頁面沒有未套樣式內容閃現或水平捲軸。
4. 英雄、海克斯與裝備原有搜尋、篩選、CRUD 權限與選取功能不退步。
5. 登入與未登入狀態皆可正常導覽。
6. `prefers-reduced-motion` 可停用非必要動畫。
7. Release build、現有 59 項測試與 HTTP smoke 皆通過。
8. 以實際瀏覽器截圖驗證桌面與手機版，不以程式碼差異代替視覺驗證。
